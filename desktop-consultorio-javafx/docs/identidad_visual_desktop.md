# Identidad visual desktop

## Propósito

Definir la identidad visual del cliente de escritorio del consultorio, incluyendo nombre ficticio, tono, personalidad, promesa visual y forma general de presentarse ante el usuario.

## Nombre ficticio asumido

### Consultorio
**Consultorio Santa Emilia**

### Software
**Santa Emilia Desktop**

## Contexto narrativo

Se asume un consultorio de barrio de clase media, profesional, cercano y ordenado. No es una clínica de lujo ni un centro hospitalario masivo. La interfaz debe verse como una herramienta real de trabajo, no como una pieza publicitaria.

## Misión visual del producto

Transmitir que el sistema es:

- claro;
- serio;
- confiable;
- ordenado;
- cómodo para trabajo prolongado.

## Visión visual del producto

Lograr una interfaz de escritorio que combine:

- la familiaridad del software clásico;
- la limpieza visual suficiente para verse actual;
- una identidad institucional discreta;
- una composición estable que no dependa de efectos exagerados.

## Tono visual

El tono general debe ser:

- institucional;
- sobrio;
- elegante sin lujo;
- clásico;
- cercano.

## Qué debe evitar la identidad

No debe sentirse:

- infantil;
- propagandística;
- exageradamente médica;
- roja de forma agresiva;
- futurista;
- minimalista vacía;
- corporativa fría al extremo.

## Paleta conceptual

### Colores predominantes
- grises;
- blanco;
- blanco grisáceo.

### Color secundario institucional
- concho de vino.

### Color puntual
- rojo solo en acentos muy controlados.

### Acento menor opcional
- naranja muy moderado para detalles concretos si hiciera falta.

## Uso conceptual de los colores

### Grises
Deben dar estructura, estabilidad y sensación de escritorio tradicional.

### Blanco y blanco grisáceo
Deben aportar respiración visual, legibilidad y limpieza.

### Concho de vino
Debe usarse como firma institucional del producto. No debe invadir toda la pantalla, pero sí marcar puntos importantes.

### Rojo
Debe usarse solo para:
- error;
- estados delicados;
- detalles médicos puntuales;
- jamás como color dominante de toda la UI.

## Branding permitido

### Sí
- logo del consultorio;
- nombre del producto en barra superior o login;
- una ilustración o imagen sutil en login;
- íconos o assets mínimos de apoyo.

### No
- llenar todo de cruces médicas;
- fondos decorativos por todas partes;
- branding repetido en cada panel;
- imágenes grandes en módulos operativos densos.

## Personalidad del shell

El shell principal debe sentirse:

- firme;
- estable;
- fijo;
- serio;
- navegable sin sorpresa.

Debe parecer una estación de trabajo del consultorio, no una vitrina publicitaria.

## Personalidad de los módulos

### Dashboard
Debe verse importante, resumido y controlado.

### Listados
Deben verse operativos, legibles y rápidos.

### Formularios
Deben verse claros, discretos y directos.

### Modales
Deben verse centrados, limpios y con jerarquía clara.

## Relación con la referencia visual compartida

Las capturas de referencia muestran una lógica acertada:

- shell fijo;
- top bar diferenciada;
- sidebar persistente;
- dashboard protagonista;
- tablas como centro del trabajo;
- modales sobrios.

La nueva identidad visual no debe romper esa lógica estructural. Solo debe reorientarla desde el azul hacia un sistema gris + vino más maduro.

## Resultado esperado

La identidad visual del desktop debe dejar una sensación inmediata de software clásico, serio y confiable, con una estética institucional contenida que haga ver al consultorio como una operación ordenada y profesional.

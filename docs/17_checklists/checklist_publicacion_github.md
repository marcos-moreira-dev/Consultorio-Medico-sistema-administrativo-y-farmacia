# Checklist publicación GitHub

## Propósito

Verificar si el repositorio está listo para publicarse o mostrarse en GitHub con una calidad mínima aceptable de orden, claridad y presentación profesional.

## Visibilidad general del repositorio

- [ ] El nombre del repositorio es claro y coherente con el proyecto.
- [ ] La estructura del repositorio no se ve caótica desde la raíz.
- [ ] El propósito del proyecto puede entenderse rápidamente al entrar.

## README y contexto inicial

- [ ] Existe un README útil en la raíz del proyecto o en el punto principal de entrada.
- [ ] El README explica qué es el sistema, qué resuelve y qué componentes tiene.
- [ ] El README no promete funcionalidades inexistentes.
- [ ] La documentación clave está ubicada y nombrada con claridad.

## Código y archivos

- [ ] No hay archivos basura, temporales o accidentales en el repositorio.
- [ ] Los archivos sensibles no se exponen.
- [ ] La configuración privada no quedó hardcodeada ni publicada sin criterio.
- [ ] La estructura de carpetas refleja el diseño del proyecto.

## Documentación técnica

- [ ] El contexto maestro del proyecto está completo.
- [ ] Dominio, arquitectura y despliegue tienen documentación base suficiente.
- [ ] Las decisiones principales del sistema se pueden rastrear en la documentación.
- [ ] Las plantillas, checklists y bloques de apoyo no se ven vacíos sin explicación.

## Calidad de presentación

- [ ] Los nombres de archivos y carpetas son consistentes.
- [ ] La redacción general es razonablemente clara.
- [ ] No hay contradicciones obvias entre documentos visibles.
- [ ] El proyecto transmite intención profesional y no simple acumulación de archivos.

## Seguridad antes de publicar

- [ ] No hay credenciales reales en el repositorio.
- [ ] No hay datos sensibles reales de pacientes ni usuarios.
- [ ] No se publican secretos, tokens o configuraciones privadas.
- [ ] Los ejemplos o seeds usan datos ficticios o controlados.

## Preparación final de portafolio

- [ ] El proyecto puede mostrarse sin explicar demasiados parches ocultos.
- [ ] Se puede describir en pocas frases su valor técnico y funcional.
- [ ] La publicación en GitHub ayuda a tu perfil profesional en lugar de debilitarlo.

## Resultado esperado

Si este checklist se cumple, el repositorio debería poder mostrarse como una pieza de trabajo seria, organizada y útil para estudio, portafolio y conversación técnica.

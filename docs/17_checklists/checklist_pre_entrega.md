# Checklist pre entrega

## Propósito

Revisar si el proyecto está en condiciones de mostrarse, compartirse o entregarse a otra persona sin transmitir desorden, contradicciones o fragilidad innecesaria.

## Presentación general

- [ ] El propósito del proyecto puede explicarse en pocas frases con claridad.
- [ ] Está clara la diferencia entre consultorio y farmacia.
- [ ] La V1 tiene un alcance comprensible y no promete más de lo que realmente hace.
- [ ] El sistema se percibe como modular y no como colección de piezas sueltas.

## Estado funcional

- [ ] Los flujos principales del consultorio funcionan.
- [ ] Los flujos principales de farmacia funcionan.
- [ ] No hay fallos obvios en la demo o recorrido principal.
- [ ] Los datos demo usados son coherentes entre sí.

## Estado técnico

- [ ] La estructura del repositorio es legible.
- [ ] Los componentes principales están identificados con claridad.
- [ ] La configuración mínima está documentada.
- [ ] El arranque local es razonablemente repetible.

## Estado documental

- [ ] El contexto maestro está completo.
- [ ] Los documentos de dominio están alineados con la arquitectura.
- [ ] Seguridad, despliegue y calidad tienen criterio definido.
- [ ] No hay contradicciones fuertes entre documentos principales.

## Estado visual y de UX

- [ ] Las superficies del sistema se distinguen claramente.
- [ ] Las pantallas no se sienten improvisadas.
- [ ] Los mensajes visibles son comprensibles.
- [ ] La UX transmite control y sobriedad.

## Seguridad y privacidad

- [ ] No se muestran datos sensibles reales.
- [ ] La parte pública no expone información privada.
- [ ] Los accesos y permisos tienen sentido mínimo.

## Cierre de entrega

- [ ] El proyecto puede mostrarse sin pedir demasiadas disculpas por caos interno.
- [ ] Lo que se enseña representa honestamente el estado real del sistema.
- [ ] Existe una narrativa clara para explicar decisiones, tradeoffs y valor del proyecto.

## Resultado esperado

Si este checklist se cumple, el proyecto ya está en condiciones razonables de ser presentado o compartido como una pieza seria de estudio, demo o portafolio.
